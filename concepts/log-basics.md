# Log Basics for SOC Analysts

## What is a log?
A log is a record of an event that happened on a system or network.

## Common log sources:
- System logs (OS events)
- Application logs
- Network logs
- Authentication logs

## Why logs matter in SOC?
Logs are the primary data source used to detect suspicious activity.

## Example:
- Multiple failed login attempts
- Access from unusual IPs

## My current understanding:
Logs help tell the story of what happened on a system.

## Confusion / To Learn:
Difference between event logs and audit logs.
